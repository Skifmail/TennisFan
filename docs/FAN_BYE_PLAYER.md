# Служебный игрок «Свободный круг» (Bye)

## Назначение

При **нечётном** числе участников или победителей в раунде один игрок не может быть спарен с другим. В таких случаях используется служебный игрок «Свободный круг» (bye).

## Когда используется

1. **R1 при нечётном числе участников (15):**
   - Сеяный 1 получает матч «игрок vs Свободный круг»
   - Матч сразу завершается walkover, игрок выходит во второй круг

2. **R2 и далее при нечётном числе победителей (7):**
   - Один победитель не имеет пары
   - **Справедливо:** bye получает сильнейший по рейтингу среди 7 победителей
   - Создаётся матч «сильнейший vs Свободный круг» (walkover)
   - Игрок выходит в полуфинал

3. **Подвал при нечётном числе проигравших R1 (7):**
   - 6 игроков играют в 3 матчах
   - 7-й (средний по списку) получает матч vs Свободный круг (walkover)
   - Все 7 проигравших R1 сыграли минимум 2 матча

4. **Финал — никогда не содержит Bye:**
   - Финал всегда играют два победителя полуфиналов
   - При каскадном создании матчей (R1→R2→R3→R4) система ждёт второй полуфинал

## Важно

**Не удаляйте** служебного игрока — он используется системой автоматически.

Если игрок был удалён, восстановите его:

```bash
python manage.py ensure_bye_player
```

## Идентификация

- Email: `bye@tennisfan.local`
- Поле `Player.is_bye = True`
- В админке отображается как «Свободный круг»
